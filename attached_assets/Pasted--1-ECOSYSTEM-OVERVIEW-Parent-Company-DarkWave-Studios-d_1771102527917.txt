
1. ECOSYSTEM OVERVIEW
Parent Company: DarkWave Studios (darkwavestudios.io)
Authentication Layer: Trust Layer (dwtl.io) — behind-the-scenes credential sync across all apps
Blockchain: Solana via Helius RPC for tamper-proof verification

Apps in the Ecosystem
App	Domain/Route	Purpose	Status
GarageBot	garagebot.io	Parts aggregator for all motorized vehicles	Live
TORQUE	garagebot.io/torque	Shop Management OS (standalone PWA)	Live
Trust Layer	dwtl.io	Cross-app identity & credential sync	Live
ORBIT	orbitstaffing.replit.app	Staffing & Payroll OS	Live (external)
Authentication Model (SSO Clarification)
Each app has its own login form — no redirect-based SSO
Trust Layer syncs credentials behind the scenes so users can use the same password everywhere
Password policy (ecosystem-wide): 8+ chars, 1 uppercase, 1 lowercase, 1 number, 1 special character

Service Worker (torque-sw.js)
Scope: /torque only — does not touch GarageBot requests
Cache names (fully isolated):
torque-v1 — app shell
torque-static-v1 — static assets
torque-dynamic-v1 — API responses
Strategy: Network-first for navigation/API, cache-first for static assets
Offline: Falls back to cached app shell
Lifecycle: skipWaiting() + clients.claim() for instant activation

Renaming TORQUE
If the name conflicts with an existing trademark, renaming takes about 30 minutes:

Find-and-replace "TORQUE"/"torque" across ~10 files
Rename public assets (manifest, SW, icons)
Update routes in App.tsx
Regenerate emblem with new name
The architecture is modular — nothing is hardcoded deep in the system
